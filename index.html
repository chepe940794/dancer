¿que significa perl?

Practical Extraction and Report Language

¿Que es Perl?

Se trata de un lenguaje de programación para extraer información de archivos de texto y generar informes apartir del contenido de los ficheros.

Perl es un lenguaje de programación interpretado, al igual que muchos otros lenguajes de Internet como Javascript o ASP. Esto quiere decir que el código de los scripts en Perl no se compila sino que cada vez que se quiere ejecutar se lee el código y se pone en marcha interpretando lo que hay escrito.

Larry Wall(Septiembre 27, 1954)

¿para que sirve?

Perl es un lenguaje visualizado para la manipulación de cadenas de caracteres, archivos y procesos.

La forma del lenguaje facilita la programación rápida y sucia, el hacer un programa rápido para que funcione. Esto hace también que se utilice para hacer prototipos rápidos de algún algoritmo que queremos ver funcionar antes que tomarnos la molestia de programarlo en un lenguaje más eficiente, como el c++. 

Ventajas

- Es facil de usar(teniendo ya conocimiento sobre el lenguaje).
- Se puede programar cualquier necesidad ya que existen llibrerias para casi cualquier cosa.
- El tiempo de desarrollo de una aplicacion es rapido.
- Disponible para gran cantidad de sistemas operativos.

Desventajas

- Dificil de aprender porque esta enfocado hacia un desarrollador que posee cierta cantidad de conocimientos sobre el lenguaje y no así hacia un estudiante que esta aprendiendo.
- El lenguaje es feo, siendo esta una de las principales razones para su difícil aprendizaje.

Dancer es un framework web escrito en Perl, inspirado originalmente por Rubi Sinatra.

Dancer es muy fácil de instalar, el número de dependencias se mantiene al mínimo.
El objetivo principal es que sea sencillo de manejar para un novato en Perl. Pero también es lo suficientemente flexible que un usuario experimentado puede lograr lo que se desea.

Algunas de sus características incluyen:

Es compatible con diferentes sistemas de serialización (JSON, XML, YAML), ideal para el desarrollo de aplicaciones REST (Representational State Transfer)
Compatibilidad con Plack y sus muchas middleware
Muchos sistemas de loggers y las plantillas disponibles.
